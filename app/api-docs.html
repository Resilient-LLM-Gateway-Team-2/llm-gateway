API Documentation

Welcome to the Resilient LLM Gateway API. This API provides intelligent routing across multiple LLM providers (OpenAI, Google Gemini) with automatic cost optimization, failover capabilities, and comprehensive observability.

🚀 Base URL: http://localhost:8000 (Development) or your production domain
📦 Response Format: All endpoints return JSON responses with consistent structure

Authentication

All API requests require authentication using an API key. Include your API key in the request headers as X-API-Key.

⚠️ Important: Never share your API key publicly. Treat it like a password.

Header Format

X-API-Key: your_api_key_here

Rate Limiting

The API implements rate limiting to ensure fair usage and service stability. Current limits are:

Plan Requests/Min Requests/Hour Daily Limit
Free 10 300 5,000
Pro 100 5,000 100,000
Enterprise Unlimited Unlimited Unlimited
Rate Limit Headers: Check response headers for current quota:
  • X-RateLimit-Limit
  • X-RateLimit-Remaining
  • X-RateLimit-Reset

POST /chat

Send a message to the LLM and get an intelligent response with cost optimization and provider routing.

POST /chat

Process a chat message with intelligent provider routing, cost optimization, and automatic failover.

Headers
Content-Type string REQUIRED
application/json
X-API-Key string REQUIRED
Your API key for authentication
Request Body
messages array[object] REQUIRED
Array of message objects with "role" and "content" fields
model string OPTIONAL
Model to use (default: gpt-3.5-turbo). Options: gpt-3.5-turbo, gpt-4, gemini-1.5-flash
Code Examples
curl -X POST http://localhost:8000/chat \ -H "Content-Type: application/json" \ -H "X-API-Key: your_api_key_here" \ -d '{ "messages": [ { "role": "user", "content": "What is 2 + 2?" } ], "model": "gpt-3.5-turbo" }'
import requests api_key = "your_api_key_here" url = "http://localhost:8000/chat" headers = { "Content-Type": "application/json", "X-API-Key": api_key } payload = { "messages": [ { "role": "user", "content": "What is 2 + 2?" } ], "model": "gpt-3.5-turbo" } response = requests.post(url, json=payload, headers=headers) print(response.json())
const apiKey = "your_api_key_here"; const url = "http://localhost:8000/chat"; const payload = { messages: [ { role: "user", content: "What is 2 + 2?" } ], model: "gpt-3.5-turbo" }; fetch(url, { method: "POST", headers: { "Content-Type": "application/json", "X-API-Key": apiKey }, body: JSON.stringify(payload) }) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error("Error:", error));

POST /budget

Set and manage monthly budget limits for specific LLM providers with warning thresholds.

POST /budget
Query Parameters
provider string REQUIRED
Provider name: "openai" or "gemini"
monthly_budget_usd number REQUIRED
Monthly budget limit in USD
warning_threshold_percent number OPTIONAL
Warning threshold as percentage (default: 80)
Code Example
curl -X POST "http://localhost:8000/budget?provider=openai&monthly_budget_usd=100&warning_threshold_percent=80" \ -H "X-API-Key: your_api_key_here"

Error Codes & Responses

The API returns standard HTTP status codes. Error responses include a detail field with more information.

Code Status Description Solution
200 OK Request successful No action needed
400 Bad Request Invalid request parameters or malformed JSON Check your request format and parameters
401 Unauthorized Missing or invalid API key Verify your X-API-Key header
403 Forbidden API key lacks required permissions Check your API key permissions
429 Too Many Requests Rate limit exceeded Wait before retrying; upgrade plan if needed
500 Internal Server Error Server error occurred Retry later or contact support
503 Service Unavailable API temporarily unavailable Check status page or retry later

Error Response Format

{ "detail": "Invalid API key provided" }

Troubleshooting

❌ "Invalid API key" error

Cause: Your API key is missing, incorrect, or has been revoked.
Solution:

  • Verify the X-API-Key header is included in your request
  • Double-check your API key spelling and case sensitivity
  • Regenerate your API key from the dashboard if needed

⏱️ "Request timeout" error

Cause: Request took too long to process.
Solution:

  • Check your internet connection
  • Try again with a shorter prompt
  • Use a model with faster response times (e.g., gpt-3.5-turbo)
  • Check the status page for service incidents

💰 "Budget exceeded" error

Cause: Your monthly budget limit for a provider has been reached.
Solution:

  • Increase your monthly budget in the dashboard
  • Switch to a cheaper provider
  • Upgrade to a higher tier plan
  • Wait until the next billing cycle

🔄 "Rate limit exceeded"

Cause: You've exceeded the request limit for your plan.
Solution:

  • Wait a few minutes before retrying
  • Implement exponential backoff in your retry logic
  • Upgrade to a higher tier plan for more requests
  • Contact support for enterprise limits
