Explore all the features that make intelligent multi-provider routing possible

Smart Routing

Intelligent provider selection based on task type, cost, and provider health. Automatic failover when primary provider fails.

  • Multi-level failover (Primary → Secondary → Fallback)
  • Health-aware routing
  • Task-optimized provider selection
  • Real-time provider status monitoring
💰

Cost Optimization

Real-time cost estimation across all providers. Monitor spending with detailed cost breakdowns and budget controls.

  • Token-based pricing models
  • Cost estimation before requests
  • Monthly spending dashboards
  • Provider cost comparisons
⚡

Request Caching

Two-layer caching system: exact match and prompt-level. Reduce latency and costs through intelligent cache hits.

  • Exact match caching (full request state)
  • Prompt-level caching (similar questions)
  • Redis-backed fast lookups
  • Automatic cache invalidation
📊

Observability

Complete request tracking, health monitoring, and analytics. Understand provider performance and costs in real-time.

  • Request logging to PostgreSQL
  • Latency tracking per provider
  • Token usage statistics
  • Route path visualization
🛡️

Budget Management

Set monthly budgets per provider with warning thresholds. Prevent unexpected costs with hard limits and alerts.

  • Per-provider budget limits
  • Warning thresholds (80%)
  • Hard limits (100%)
  • Real-time budget monitoring
🔑

API Key Management

Secure authentication with API keys. Track usage per key and manage access control at the application level.

  • Unique API key per application
  • Per-key usage tracking
  • Session-based authentication
  • Secure credential handling
