Project Resources

Complete documentation of features, stack, and architecture

✓ Core Features

Multi-provider routing - Route requests across OpenAI, Gemini, and fallbacks
Automatic failover (3-level) - Primary → Secondary → Fallback
Request retry with backoff - Exponential backoff on failures
Real-time cost estimation - Token-based pricing calculations
Two-layer request caching - Exact match + prompt-level caching
Budget management & limits - Monthly per-provider budgets with warnings

📊 Observability

Request logging - PostgreSQL request history
Cost tracking - Per-request and monthly cost aggregation
Route path visualization - See which providers were used
Health checks - API, Database, and Cache status
Request latency tracking - Performance metrics per request
Token usage statistics - Input and output token counts

🔐 Security & Auth

Session-based authentication - Secure user login with tokens
API key management - Unique keys per application
HTTPOnly cookies - Protection against XSS attacks
CSRF protection - Secure form submissions
Password hashing - SHA256 secure credential storage
7-day session expiration - Automatic session cleanup

Request Flow Architecture

User Request

↓

FastAPI Router (Endpoint validation & auth)

↓

Cache Check (Redis - 2 layers)

↓

Provider Router (Select: OpenAI → Gemini → Fallback)

↓

API Call (With retry & backoff)

↓

Response Caching (Store in Redis)

↓

Cost Calculation (Token-based pricing)

↓

Logging (PostgreSQL + Observability)

↓

Response to User

Key Metrics & Performance

⚡ Caching Performance

Exact match hit: <5ms response time
Prompt-level hit: <10ms response time
Cache miss (API call): 500-2000ms (varies by provider)
Typical cache hit rate: 20-40% with repeat questions

💰 Cost Savings

Using Gemini only: 70% cheaper than OpenAI
With intelligent routing: 40-60% cost reduction
Cache hit savings: 100% (no API charge)
Budget control: Prevent overspending with hard limits

🛡️ Reliability

Uptime guarantee: 99.9% with fallback provider
Automatic failover: Instant switch on primary failure
Retry mechanism: Exponential backoff (max 3 retries)
Health monitoring: Real-time status of all components
